## 2. 검색 기반 플로우

```

    │
    ▼

    │

    │

    ▼

    │
    ├─→ 결과 있음 → 원하는 아이콘 선택
    └─→ 결과 없음 → 다른 키워드로 재검색
    │
    ▼
[기본 사용 플로우 계속]
```

## 사용자 시나리오

### 시나리오 1: 마케팅 매니저

**배경**: 회사 프레젠테이션 준비 중

**목표**: 브랜드 컬러로 통일된 아이콘 필요

**플로우**:
1. Icony 접속
2. "chart" 검색 → 차트 아이콘 찾기
3. 회사 브랜드 컬러 (#2563EB) 적용
4. 128px 크기 선택
5. 클립보드에 복사
6. PowerPoint에 붙여넣기
7. 다음 아이콘 반복 (3개 아이콘 필요)

**소요 시간**: ~3분

**만족도 포인트**:
- ✅ 빠른 검색
- ✅ 정확한 브랜드 컬러 적용
- ✅ 클립보드 복사로 빠른 삽입

---

### 시나리오 2: 프론트엔드 개발자

**배경**: 웹 앱 UI 개발 중 임시 아이콘 필요

**목표**: 프로토타입용 아이콘 빠르게 획득

**플로우**:
1. Icony 접속
2. "settings" 검색
3. 첫 번째 아이콘 선택 (빠른 결정)
4. 기본 색상(#000000) 유지
5. 64px 선택 (웹용 적정 크기)
6. PNG 다운로드
7. `public/icons/` 폴더에 저장
8. 코드에서 이미지 참조

**소요 시간**: ~1분

**만족도 포인트**:
- ✅ 빠른 검색
- ✅ 간단한 다운로드
- ✅ 별도 디자인 툴 불필요

---

### 시나리오 3: UI/UX 디자이너

**배경**: 디자인 시스템 아이콘 세트 구축

**목표**: 일관된 스타일의 아이콘 20개 다운로드

**플로우**:
1. Icony 접속
2. 필요한 아이콘 리스트 확인
3. 첫 번째 아이콘 선택 (예: "home")
4. 디자인 시스템 컬러 적용 (#6366F1)
5. 256px 선택 (고해상도)
6. 다운로드
7. 다음 아이콘 선택 시 색상/크기 유지됨
8. 20개 아이콘 반복 다운로드
9. Figma에 import

**소요 시간**: ~10분

**만족도 포인트**:
- ✅ 색상/크기 설정 유지
- ✅ 일관된 스타일
- ✅ 고품질 PNG

## 인터랙션 디테일

### 아이콘 선택

**시각적 피드백**:
```
기본 상태:
┌──────────┐
│          │  ← 회색 테두리
│   Icon   │
│          │
└──────────┘

호버 상태:
┌──────────┐
│          │  ← 파란색 테두리
│   Icon   │  ← 살짝 확대
│          │
└──────────┘

선택 상태:
┌──────────┐
│          │  ← 진한 파란색 테두리
│   Icon   │  ← 파란색 배경
│          │
└──────────┘
```

### 색상 변경

**실시간 미리보기**:
```
스와치 클릭:
  클릭 → 즉시 색상 변경 (0ms 딜레이)

컬러 피커 드래그:
  드래그 중 → 실시간 색상 변경 (debounce 없음)
  마우스 업 → 최종 색상 확정
```

### 내보내기

**다운로드 프로세스**:
```
[Download PNG] 버튼 클릭
    ↓
로딩 스피너 표시 (~500ms)
    ↓
브라우저 다운로드 시작
    ↓
성공 토스트 알림 (2초)
    ↓
버튼 정상 상태 복귀
```

**클립보드 복사 프로세스**:
```
[Copy to Clipboard] 버튼 클릭
    ↓
로딩 스피너 표시 (~300ms)
    ↓
클립보드에 이미지 복사
    ↓
성공 토스트 알림 (2초)
    ↓
버튼 정상 상태 복귀

에러 발생 시:
    ↓
에러 토스트 알림 (3초)
"Clipboard permission denied"
```

## 엣지 케이스

### 1. 브라우저 미지원 (Safari < 13.1)

**플로우**:
```
클립보드 복사 시도
    ↓
브라우저 미지원 감지
    ↓
에러 메시지 표시
"Your browser doesn't support clipboard operations.
Please use Download instead."
    ↓
사용자는 다운로드 버튼 사용
```

### 2. 검색 결과 없음

**플로우**:
```
검색어 입력: "xyz123"
    ↓
매칭 결과 없음
    ↓
빈 상태 메시지 표시
"No icons found for 'xyz123'"
"Try different keywords or browse all icons"
    ↓
사용자 액션 옵션:
├─→ 다른 키워드 시도
└─→ 검색 초기화 버튼 클릭
```

### 3. 아이콘 미선택 상태에서 내보내기

**플로우**:
```
내보내기 버튼 클릭
    ↓
선택된 아이콘 없음 확인
    ↓
경고 토스트 표시
"Please select an icon first"
    ↓
(버튼은 비활성화되어 있어야 함)
```

## 성능 플로우

### 초기 로딩

```
페이지 접속
    ↓
HTML 로드 (100ms)
    ↓
CSS 로드 (200ms)
    ↓
JS 번들 로드 (500ms)
    ↓
React 하이드레이션 (300ms)
    ↓
아이콘 데이터 로드 (100ms)
    ↓
첫 번째 렌더링 완료
    ↓
Total: ~1.2초
```

### 아이콘 검색

```
키보드 입력 시작
    ↓
Debounce 대기 (300ms)
    ↓
검색 필터 실행 (<50ms)
    ↓
리렌더링 (100ms)
    ↓
결과 표시 완료
```

### PNG 생성

```
내보내기 버튼 클릭
    ↓
SVG → Canvas 변환 (200ms)
    ↓
Canvas → PNG Blob (100ms)
    ↓
다운로드/클립보드 처리 (100ms)
    ↓
Total: ~400ms
```

## 모바일 사용 플로우

### 터치 인터랙션

```
모바일 사용자 접속
    ↓
세로 스택 레이아웃 표시
    ↓
위: 검색 및 아이콘 갤러리

스와이프하여 아이콘 탐색
    ↓
탭하여 아이콘 선택
    ↓
아래로 스크롤하여 커스터마이징
    ↓
```

**터치 타겟**:
- 최소 44x44px (Apple HIG)
- 버튼 간격: 8px 이상

## 사용자 만족도 극대화 요소

### 1. 즉각적인 피드백
- 모든 액션에 200ms 이내 시각적 피드백
- 로딩 상태 명확히 표시
- 성공/실패 알림

### 2. 직관적인 인터페이스
- 3번의 클릭으로 목표 달성
- 명확한 라벨링
- 일관된 디자인 패턴

### 3. 유연한 워크플로우
- 순서 자유로움 (색상 먼저 또는 아이콘 먼저)
- 실시간 미리보기
- 언제든 변경 가능

### 4. 성능 최적화
- 빠른 초기 로딩
- 부드러운 인터랙션
- 최소한의 대기 시간
